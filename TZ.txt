Общая цель:
Разработать систему копи-трейдинга, которая автоматически повторяет все сделки мастер-аккаунта на пуле донорских аккаунтов через биржевое API (uid), с управлением через Telegram-бота.

Что именно копируем:

Рыночные ордера (маркет buy/sell).

Лимитные ордера (открытие/закрытие).

Стоп-лоссы и тейк-профиты (триггерные ордера).

Отмена ордеров мастера должна синхронно отменять соответствующие ордера на донорских аккаунтах.

Рандомизация для антибана

Лёгкий рандомайз объёма:

мастер, например: 59 USDT

доноры: 60.1, 59.4, 61.4 и т.д. (небольшой разброс в заданном диапазоне).

Опциональный рандомайз задержки между исполнением у мастера и отправкой ордеров на доноров:

параметр «доп.задержка», мс (0 = без доп.задержки, только технический минимум).


Скорость и задержки

Цель — максимально возможная скорость копирования.

Ориентир: техническая задержка порядка 0.2–0.5 сек между мастером и донорами (фактическое значение зависит от биржи, ликвидности и количества аккаунтов).

Фиксация сделок мастера — по API (по WebSocket/stream), последующее копирование — асинхронно на доноров.

Архитектура аккаунтов

Базовая конфигурация: 1 мастер → до 25 донорских аккаунтов.

В Telegram-боте — быстрый выбор количества задействованных аккаунтов из пула:

кнопки вида: «ВКЛ 5», «ВКЛ 10», «ВКЛ 15», и т.д.

либо выбор диапазонов: например, аккаунты 2–7, 9–11. -- это.

На будущее: возможность иметь несколько мастер-аккаунтов, каждый со своей группой доноров (1→5–10, 2→6–9 и т.п.). --
-- возможно будет проще сделать через отдельного бота ТГ если таких мастер-аккаунтов немного.

Масштабирование объёма сделок

Для каждого донорского аккаунта — коэффициент копирования:

например: 0.5x, 1x, 2x относительно объёма мастера.

Коэффициент учитывается при расчёте суммы/количества ордера на доноре.


Прокси и сетевое соединение

Поддержка двух режимов:

с прокси;

без прокси (для тестов или по выбору пользователя).

Принцип: 1 аккаунт = 1 сессия, прокси между аккаунтами не ротируются.

Для каждого аккаунта — список прокси:

1 основной + 1–2 резервных.

При «слёте» прокси — автопереключение на следующее из списка для данного аккаунта.


Telegram-бот:

Авторизация и доступ -- по ID админа

Админ управляет списком разрешённых пользователей (добавление/удаление ID прямо из бота).
